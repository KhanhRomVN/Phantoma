# Account Module Styling Rules

## ⚠️ QUAN TRỌNG: `var(--*)` LÀ DẠNG LỖI TRONG CẢ INLINE STYLES VÀ CLASSNAME

### Quy tắc cốt lõi:
1. **`var(--*)` trong className là SAI** — không có biến CSS nào được định nghĩa trong tailwind.config.js với tên đó. Ví dụ: `text-[var(--primary-text)]` là sai vì biến đúng là `--text-primary`, không phải `--primary-text`.

2. **`var(--*)` trong inline styles cũng là SAI** — không thể lấy giá trị CSS variable trong React inline styles. Phải dùng `$('--variable')` (import từ `@renderer/utils/color`).

3. **Chỉ được dùng các biến đã được định nghĩa trong tailwind.config.js**:
   - `--text-primary` → `text-text-primary`
   - `--text-secondary` → `text-text-secondary`
   - `--text-foreground` → `text-text-foreground`
   - `--primary` → `text-primary` / `border-primary` / `bg-primary`
   - `--error` → `text-error` / `bg-error` / `border-error`
   - `--success` → `text-success` / `bg-success` / `border-success`
   - `--warn` → `text-warn`
   - `--teal` → `text-teal` / `border-teal` / `bg-teal`
   - `--info` → `text-info` / `border-info` / `bg-info`
   - `--violet` → `text-violet` / `border-violet` / `bg-violet`
   - `--purple` → `text-purple` / `border-purple` / `bg-purple`
   - `--card-background` → `bg-card-background`
   - `--input-background` → `bg-input-background`
   - `--border` → `border-border`
   - `--background` → `bg-background`
   - `--accent` → KHÔNG có class → dùng `$('--accent')` trong inline styles

### Cách sửa:
- **Trong className**: dùng class tương ứng từ mapping trên
- **Trong inline styles**: dùng `$('--variable')`
- **KHÔNG BAO GIỜ dùng `var(--*)`** — luôn chuyển thành class hoặc `$('--variable')`

## Quy tắc chuyển đổi CSS

Thứ tự ưu tiên khi styling:
1. **className** (Tailwind) — ưu tiên cao nhất
2. **`$('--variable')`** — dùng trong inline styles khi không thể dùng className
3. **KHÔNG dùng `var(--variable)`** — đây là dạng lỗi, phải chuyển thành class hoặc `$('--variable')`

## Rules for AI

### Quy tắc bắt buộc:
1. **`var(--text-primary)` trong className** → chuyển thành `text-text-primary`
2. **`var(--text-secondary)` trong className** → chuyển thành `text-text-secondary`
3. **`var(--)` trong inline styles** → chuyển thành `$('--variable')`
4. **`var(--)` trong className** → chuyển sang class tương ứng theo mapping trên
5. **`$('--variable')` trong inline styles** → kiểm tra xem có class tương ứng không, nếu có thì chuyển sang className
6. **Animation** → giữ nguyên inline style (vì có keyframes tùy chỉnh)

### Exception Cases
- `bg-[var(--error,rgba(239,68,68,0.08))]` — giữ nguyên vì có fallback và alpha
- `text-[var(--success,#22c55e)]` — chuyển sang `text-success` nếu không cần fallback
- `bg-[var(--sidebar-item-hover,rgba(99,102,241,0.08))]` — giữ nguyên vì có fallback

## 🆕 QUY TẮC BỔ SUNG: `var(--*)` trong className với fallback

**Khi gặp `var(--*)` trong className với fallback (ví dụ: `border-[var(--teal,rgba(...))]`):**
- **KHÔNG cần giữ fallback**
- **Chuyển trực tiếp sang class Tailwind tương ứng**
- Ví dụ: `border-[var(--teal,rgba(168,85,247,0.4))]` → `border-teal`
- Ví dụ: `text-[var(--info,#0ea5e9)]` → `text-info`
- Ví dụ: `bg-[var(--violet,#8b5cf6)]` → `bg-violet`

**Lý do:** Các biến CSS đã được định nghĩa trong `tailwind.config.js` với màu sắc cụ thể, không cần fallback trong className.
